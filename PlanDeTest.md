Liste des exigences
===============
- TP. 1: L’utilisateur a comme choix d’entrer `—english` comme premier paramètre si celui-ci veut que le programme soit en anglais.
- TP. 2: L’utilisateur doit obligatoirement entrer en ligne de commande le chemin du fichier d’entrée et le chemin du fichier de sortie.
- TP. 3: Cette étape est la première dans le cas où l’utilisateur
  garde le programme français, celle-ci devient la deuxième en cas contraire.
- TP. 4: Un fichier de sortie sera alors créé. Pour afficher le contenu de celui-ci,
  l’utilisateur doit utiliser la commande `cat` suivit du chemin du fichier de sortie. 

| N. | Fonctionnalité                                            | Résultat attendu                                                                                                                 | Description                                                                                                                                                                                        | Données                                                                                       |
|----|-----------------------------------------------------------|----------------------------------------------------------------------------------------------------------------------------------|----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|-----------------------------------------------------------------------------------------------|
| 1  | Test de champ manquant                                    | Lance une exception `ChampVideDansFichierEntree` et affiche un un message d'erreur approprié                                     | Vérifiez que le programme génère une exception et se termine de manière appropriée en affichant le message d'erreur correspondant en cas de champ manquant dans le fichier d'entrée.               | Un fichier d'entree csv avec un champ manquant à une ligne spécifique.                        |
| 2  | Test du nombre de champs inattendus                       | Lance une exception `ArrayIndexOutOfBounds` et affiche un message d'erreur approprié                                             | Vérifiez que le programme génère une exception et se termine de manière appropriée en affichant le message d'erreur correspondant en cas de nombre de colonnes incorrect dans le fichier d'entrée. | Un fichier avec un nombre de colonnes en trop ou en moins.                                    |
| 3  | Test de l'arrondissement invalide                         | Lance une exception `InformationInvalideDansLeFichier` et affiche un message d'erreur approprié                                  | Vérifiez que le programme génère une exception et se termine de manière appropriée en affichant le message d'erreur correspondant en cas un arrondissement invalide dans le fichier d'entrée.      | Un fichier avec un arrondissement non reconnu par le fichier json fourni.                     |
| 4  | Test de la description d'intervention invalide            | Lance une exception `InformationInvalideDansLeFichier` et affiche un message d'erreur approprié                                  | Vérifiez que le programme génère une exception et se termine de manière appropriée en affichant le message d'erreur correspondant en cas une description invalide dans le fichier d'entrée.        | Un fichier avec une description non valide selon le fichier json fourni.                      |
| 5  | Test du format d'heure invalide                           | Lance une exception `InformationInvalideDansLeFichier` et affiche un message d'erreur approprié                                  | Vérifiez que le programme génère une exception et se termine de manière appropriée en affichant le message d'erreur correspondant en cas d'un format d'heure invalide dans le fichier d'entrée.    | Un fichier avec une heure d'intervention au format incorrect, qui n'est pas ISO 8601 (HH:MM). |
